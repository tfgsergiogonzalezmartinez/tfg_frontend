Introducción

El objetivo principal del creador de páginas web es facilitar la creación de proyectos a través de plantillas predefinidas. Esto es ideal para personas que no tienen conocimientos de programación, ya que la aplicación se encarga de generar todo el proyecto junto con los ficheros necesarios para el despliegue utilizando Docker. El usuario simplemente tiene que ejecutar un comando docker-compose up -d y ya tendrá su página web en funcionamiento. A continuación, se detallan las funcionalidades clave de esta aplicación:

Seguridad JWT

La aplicación está protegida mediante seguridad JWT (JSON Web Token) para la autenticación de usuarios. Esto asegura que solo los usuarios autenticados pueden acceder a los endpoints protegidos. A continuación, se describen las principales características de la seguridad JWT en la aplicación:

  • Protección de Endpoints: Todos los endpoints críticos de la aplicación están protegidos mediante JWT, garantizando que solo los usuarios autenticados pueden acceder a ellos.
  • Autenticación y Autorización: La autenticación se realiza mediante el envío de un token JWT con cada solicitud, el cual es verificado por el servidor para asegurar que el usuario está autorizado para acceder al recurso solicitado.
  • Gestión de Roles: Los roles de los usuarios se gestionan a través de JWT. Cada token incluye información sobre el rol del usuario (por ejemplo, usuario, administrador), lo que permite al servidor controlar el acceso a diferentes recursos y funcionalidades según el rol asignado.
  • Expiración de Tokens: Los tokens JWT tienen un tiempo de expiración, lo que añade una capa adicional de seguridad al requerir a los usuarios autenticarse nuevamente después de un periodo determinado.

Administración

La aplicación permite a los usuarios con roles de administrador gestionar los roles de otros usuarios. Esta funcionalidad es crucial para mantener un control adecuado sobre el acceso y las capacidades dentro de la aplicación. A continuación, se detallan los pasos que los administradores pueden seguir para modificar los roles de los usuarios:

Modificación de Roles de Usuarios

Los administradores pueden modificar los roles de los usuarios a través del panel de administración. A continuación se describen los pasos para realizar esta tarea:

  1. Acceso al Panel de Administración: Los administradores deben iniciar sesión y acceder al panel de administración, disponible en la navegación principal.
  2. Gestión de Usuarios: Dentro del panel de administración, hay una tabla específica para la gestión de usuarios. Aquí, los administradores pueden ver todos los usuarios registrados en la aplicación.
  3. Modificación del Rol: En la vista de la tabla de usuarios, los administradores pueden cambiar el rol del usuario mediante un selector.
  4. Guardado de Cambios: Después de seleccionar el nuevo rol, el administrador debe guardar los cambios para que estos se apliquen. Esto se puede hacer mediante el botón de "Guardar".
La gestión de roles de usuarios es una funcionalidad esencial para garantizar la seguridad y el control de acceso en la aplicación. Los administradores pueden modificar los roles de los usuarios según sea necesario para mantener un entorno seguro y eficiente.
Chat en Vivo con WebSockets

La aplicación incluye un sistema de chat en vivo que utiliza WebSockets para la comunicación en tiempo real. Este chat está protegido mediante tokens JWT para asegurar que solo los usuarios autenticados pueden participar en las conversaciones. A continuación, se describen las características principales del chat en vivo:

  • Historial de Mensajes: Los usuarios pueden ver el historial de mensajes de las conversaciones, lo que les permite revisar mensajes anteriores en cualquier momento.
  • Notificaciones de Mensajes no Leídos: Los usuarios reciben notificaciones cuando tienen mensajes no leídos. Cuando se conectan y han recibido mensajes anteriormente, el chat aparecerá abierto y mostrará el número de mensajes sin leer.
  • Conversaciones Activas: Los usuarios pueden ver todas sus conversaciones activas, facilitando la gestión y seguimiento de las mismas.
Soporte en Vivo

La aplicación dispone de una ventana de soporte, donde los usuarios pueden abrir tickets con sus problemas. También se incluye un chat en vivo por WebSockets para comunicarse con los administradores de soporte. A continuación, se describen las características principales del sistema de soporte:

  • Apertura de Tickets: Los usuarios pueden abrir tickets describiendo sus problemas o consultas. Estos tickets son visibles para todos los administradores.
  • Notificaciones de Soporte: Cuando un administrador recibe una notificación de soporte, todos los administradores son notificados. Los administradores pueden ver quién es el usuario, pero el usuario no sabe quién es el administrador; se muestra un avatar genérico de soporte.
  • Cierre de Tickets: Una vez que se ha solucionado un problema, el administrador puede cerrar el ticket, lo que lo elimina de la ventana de soporte tanto para el usuario como para los administradores.
Generación de Proyectos

Los usuarios pueden crear proyectos seleccionando una plantilla. La plantilla pedirá un conjunto de datos para generar sus modelos de objetos. A continuación, se describen los pasos principales en el proceso de generación de proyectos:

  • Importación de Datos: Los usuarios pueden importar cualquier tipo de CSV, separados por comas los campos. Los arrays deben estar formateados como "item1,item2,item3", separados por comas.
  • Enlace de Tipos de Datos: No es necesario que los CSV importados tengan el mismo número de campos. El siguiente paso en la generación de la web app es enlazar los tipos de datos del usuario con los de la plantilla. Esto permite que el generador cree la base de datos con los datos indicados.
Personalización de Proyectos

Los usuarios pueden personalizar sus proyectos eligiendo un logo para su aplicación y los colores principales de la misma. En el caso de utilizar la plantilla de tienda, se podrá escoger la moneda de los precios. A continuación, se describen los pasos principales en la personalización de proyectos:

  • Selección de Logo: Los usuarios pueden subir un logo personalizado para su aplicación, lo que les permite dar una identidad visual única a su proyecto.
  • Configuración de Colores: Los usuarios pueden elegir los colores principales de su aplicación, lo que les permite personalizar la apariencia visual de su proyecto.
  • Selección de Moneda: En el caso de utilizar la plantilla de tienda, los usuarios pueden seleccionar la moneda en la que se mostrarán los precios, lo que les permite adaptar su tienda a diferentes mercados.
  • Generación del Proyecto: Una vez completada la configuración, los usuarios pueden pulsar en Generar Proyecto y descargarán su proyecto generado. El único paso a seguir es descomprimir el archivo y ejecutar docker compose up -d. Luego, se podrá acceder desde localhost:4200 al frontend y desde localhost:5000 al backend.
Plantilla: Tienda

La plantilla Tienda está diseñada para generar automáticamente una plataforma de comercio electrónico completa, permitiendo a los usuarios visualizar y adquirir productos de manera sencilla y eficiente. A continuación, se describen las características y funcionalidades principales de esta plantilla:

Generación Automática de Items de Venta

Al utilizar la plantilla Tienda, se generarán automáticamente todos los ítems de venta, incluyendo:

  • Nombre del Producto
  • Precio
  • Tallas disponibles
  • Colores disponibles
  • Descripción
  • Imágenes
  • Categoria del producto

Una vez que la tienda esté en funcionamiento, los usuarios podrán añadir más ítems de venta directamente desde la aplicación, facilitando la gestión y expansión del inventario.

Importación de Categorías desde CSV

Además de los productos, las categorías también se pueden importar a partir de un archivo .csv. Cada categoría importada puede ser relacionada con los productos a través del campo categoría, permitiendo dividir y filtrar la tienda en diferentes secciones. Esto facilita la organización y la navegación dentro de la tienda, mejorando la experiencia del usuario.

Acceso Público a la Página Web

La página web generada a partir de la plantilla Tienda estará accesible para todo el mundo sin necesidad de iniciar sesión. Esto permite que cualquier visitante pueda ver los productos disponibles y sus detalles, fomentando la exploración y el interés en realizar compras.

Creación de Cuenta para Realizar Pedidos

Para realizar un pedido, los usuarios deberán crear una cuenta en la plataforma. Este proceso de registro es sencillo y permite a los usuarios acceder a funcionalidades adicionales, como:

  • Realizar pedidos
  • Guardar información de contacto y envío
  • Acceder al historial de pedidos
Carrito de Compras

La aplicación incluye un carrito de compras donde los usuarios pueden ver y gestionar todos los ítems que han añadido a su cesta. El carrito de compras permite a los usuarios:

  • Revisar los productos seleccionados
  • Eliminar ítems
  • Proceder al pago
Sección "Pedidos"

Los usuarios pueden ver sus pedidos en la sección del header llamada "Pedidos". Aquí pueden revisar el estado de sus pedidos, historial de compras y detalles de cada pedido realizado.

Administración de Pedidos

Los usuarios con rol de administrador tienen acceso a una vista completa de todos los pedidos realizados por todos los usuarios. Esto les permite gestionar eficientemente las órdenes y asegurar una operación fluida de la tienda.

Personalización de la Página

La plantilla Tienda también permite una personalización extensiva de la página web, incluyendo:

  • Colores de la página
  • Moneda utilizada para los precios
  • Logo de la página
  • Título de la página

Estas opciones de personalización aseguran que cada tienda pueda reflejar la identidad visual y preferencias de su propietario.

Seguridad por Token JWT

La aplicación cumple con los estándares de seguridad mediante el uso de tokens JWT para la autenticación de usuarios. Esto asegura que todas las operaciones sensibles, como la realización de pedidos y la administración de la tienda, estén protegidas y accesibles solo para usuarios autorizados.

La plantilla Tienda ofrece una solución completa y personalizable para la creación de una plataforma de comercio electrónico, asegurando una experiencia de usuario fluida y segura tanto para clientes como para administradores.
Introducción a Docker

Docker es una plataforma que permite desarrollar, enviar y ejecutar aplicaciones dentro de contenedores. Los contenedores son entornos ligeros y portátiles que pueden incluir todo lo necesario para ejecutar una aplicación: código, runtime, herramientas de sistema, librerías y configuraciones. A continuación, se describen las principales características de Docker:

  • Portabilidad: Los contenedores Docker pueden ejecutarse en cualquier sistema que tenga Docker instalado, sin importar las configuraciones del entorno local. Esto facilita la portabilidad de las aplicaciones entre diferentes entornos de desarrollo, prueba y producción.
  • Consistencia: Los contenedores Docker garantizan que las aplicaciones se ejecuten de manera consistente en cualquier entorno, ya que incluyen todas las dependencias y configuraciones necesarias.
  • Aislamiento: Los contenedores Docker proporcionan un aislamiento a nivel de proceso, lo que permite ejecutar múltiples contenedores en el mismo host sin interferencias entre ellos.
  • Escalabilidad: Docker facilita la escalabilidad de las aplicaciones al permitir el despliegue de múltiples contenedores en un clúster, utilizando herramientas como Docker Swarm o Kubernetes.
Diferencias entre Docker y Máquinas Virtuales

Aunque Docker y las máquinas virtuales (VMs) pueden parecer similares, tienen diferencias clave que los hacen adecuados para diferentes casos de uso. A continuación, se describen las principales diferencias entre Docker y las máquinas virtuales:

  • Uso de Recursos: Docker utiliza contenedores que comparten el mismo sistema operativo del host, lo que los hace más ligeros y rápidos de iniciar que las VMs, que requieren un sistema operativo completo por cada instancia. Esto reduce significativamente el uso de recursos y el tiempo de inicio.
  • Portabilidad: Los contenedores Docker pueden ejecutarse en cualquier sistema que tenga Docker instalado, sin importar las configuraciones del entorno local. Esto facilita la portabilidad de las aplicaciones, ya que no dependen del sistema operativo subyacente.
  • Aislamiento: Aunque las VMs ofrecen un aislamiento completo al ejecutar un sistema operativo completo en cada instancia, los contenedores Docker proporcionan un aislamiento a nivel de proceso. Esto es suficiente para la mayoría de las aplicaciones y es más eficiente en términos de recursos.
  • Velocidad: Los contenedores Docker se inician en cuestión de segundos, mientras que las VMs pueden tardar varios minutos en iniciar debido a la carga del sistema operativo completo.
  • Almacenamiento: Docker utiliza una capa de almacenamiento que permite compartir capas comunes entre diferentes contenedores, lo que reduce significativamente el tamaño total de almacenamiento. Las VMs, por otro lado, requieren almacenamiento completo para cada sistema operativo.
Despliegue con Docker

El despliegue de aplicaciones con Docker es un proceso sencillo que implica la creación de contenedores a partir de imágenes Docker. A continuación, se describen los pasos necesarios para desplegar tu proyecto utilizando Docker:

  1. Preparación del Proyecto: Descomprime el archivo descargado de tu proyecto generado. Asegúrate de que todos los archivos necesarios estén en el directorio del proyecto.
  2. Navegación al Directorio del Proyecto: Abre una terminal y navega hasta el directorio del proyecto descomprimido utilizando el comando cd. Por ejemplo: cd /ruta/al/proyecto.
  3. Ejecución del Comando Docker: Ejecuta el comando docker compose up -d para iniciar los contenedores en segundo plano. Este comando lee el archivo docker-compose.yml en el directorio del proyecto y crea y arranca los contenedores definidos en ese archivo.
  4. Acceso a la Aplicación: Una vez que los contenedores estén en funcionamiento, podrás acceder al frontend de tu aplicación desde localhost:4200 y al backend desde localhost:5000. Estos puertos pueden variar según la configuración de tu archivo docker-compose.yml.
  5. Monitoreo y Gestión: Puedes monitorear y gestionar tus contenedores utilizando comandos Docker adicionales, como docker ps para listar los contenedores en ejecución, docker logs [nombre_del_contenedor] para ver los logs de un contenedor específico, y docker stop [nombre_del_contenedor] para detener un contenedor.
Resumen

Esta aplicación proporciona una solución completa para la creación y despliegue de proyectos web a través de plantillas predefinidas, con soporte integrado y opciones de personalización. La utilización de tecnología Docker simplifica el proceso de despliegue, garantizando que los entornos sean consistentes y portátiles. Además, incluye un sistema de chat en vivo que utiliza WebSockets para la comunicación en tiempo real, protegido mediante tokens JWT. Con esta documentación, podrás aprovechar todas las funcionalidades de la aplicación y desplegar tus proyectos de manera eficiente y segura.

Espero que esta documentación te haya proporcionado una comprensión clara de cómo utilizar la aplicación y desplegar tus proyectos utilizando Docker.
